# AI 협업 사이클 워크플로우

이 워크플로우는 기능 1개 단위로 실행되는 Claude-ChatGPT 협업 사이클이다.

## 사전 조건
- `_ai-collab/` 디렉토리에 5개 문서가 모두 존재
- PROJECT_PLAN.md, SPEC.md, UI_SPEC.md가 프로젝트에 맞게 채워져 있음

## 사이클 단계

### Step 1: 작업 선택 및 문서 업데이트 (Claude)
1. TASKS.md에서 다음 작업(P0 우선)을 선택
2. 해당 작업 상태를 🔨(진행 중)으로 변경
3. 필요시 SPEC.md, UI_SPEC.md 업데이트 (해당 작업 관련 섹션)
4. TASKS.md에 작업 상세(요구사항, DoD, 검증 체크리스트) 작성

### Step 2: ChatGPT 프롬프트 생성 (Claude)
1. PROMPTS.md의 **구현 명령 템플릿** 기반으로 프롬프트 작성
2. 해당 작업에 필요한 SPEC/UI_SPEC 섹션 발췌 포함
3. 사용자에게 프롬프트 전달

### Step 3: ChatGPT에 전달 (사용자)
1. ChatGPT 대화가 새로 시작된 경우: **고정 규칙 프롬프트** 먼저 전달
2. **스펙 전달 템플릿**으로 관련 스펙 전달
3. **구현 명령** 전달

### Step 4: 구현 산출물 수령 (사용자)
1. ChatGPT의 산출물을 수령
2. 파일들을 프로젝트에 적용 (복사/붙여넣기 또는 저장)

### Step 5: 결과 보고 (사용자 → Claude)
1. 아래 형식으로 Claude에게 보고:
```
## 작업 결과 보고: [작업명]
### 변경 파일
- (파일 목록)
### 핵심 변경
- (변경 요약)
### 실행 여부
- 빌드: 성공/실패
- 실행: 성공/실패
### ChatGPT 질문/이의
- (있으면 기재)
```

### Step 6: 냉정 검수 (Claude)
1. 변경된 파일들을 직접 확인
2. SPEC.md/UI_SPEC.md 대비 준수 여부 검증
3. 검증 체크리스트 실행 지시 (사용자가 로컬에서 실행)
4. Must/Should/Nice 수정 지시 작성 (PROMPTS.md 패치 템플릿 사용)

### Step 7-8: 수정 사이클 (반복)
1. 사용자: 수정 지시를 ChatGPT에 전달
2. ChatGPT: diff 패치 + 회귀 확인 제출
3. 사용자: 결과를 Claude에 보고
4. Must가 남아있으면 Step 6으로 반복

### Step 9: 완료 판정 (Claude)
1. 모든 Must 항목 해결 확인
2. TASKS.md에서 작업 상태를 ✅(완료)로 변경
3. 다음 작업으로 Step 1 반복

## 주의사항
- Claude와 ChatGPT는 대화를 공유할 수 없다. 문서가 유일한 기준이다.
- 스펙 변경은 반드시 Claude가 문서를 업데이트한 후 진행한다.
- ChatGPT 대화가 길어지면 새 대화 시작 + 고정 규칙 재전달.
