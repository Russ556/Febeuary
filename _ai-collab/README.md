# AI 협업 운영 총괄 가이드

> **절대 전제**: Claude와 ChatGPT는 서로의 대화 내역을 공유/열람할 수 없다.  
> 사용자가 문서와 프롬프트를 복사·붙여넣기로 전달하는 **연결자** 역할을 한다.

---

## 1. 역할 정의

| 역할 | 담당자 | 핵심 책임 |
|------|--------|-----------|
| **리드(Lead)** | Claude | 설계 · 규격 · 검수 · 지휘 · 디자인 의사결정 |
| **빌더(Builder)** | ChatGPT | 구현 · 통합 · 디버깅 · 테스트 |
| **연결자(Connector)** | 사용자 | 문서/프롬프트 전달 · 산출물 요약 전달 · 로컬 실행 검증 |

### 1.1 Claude 행동 원칙
- **냉정·비판적 검토**: "그럴듯한 답변"을 신뢰하지 않는다.
- 검증이 필요한 부분은 반드시 **재현 가능한 검증 절차**(커맨드, 테스트, 시나리오)를 설계한다.
- 디자인 의사결정(레이아웃/CSS/UI)은 **Claude가 주도**한다.
- 스펙 변경 권한은 **Claude에게만** 있다.

### 1.2 ChatGPT 행동 원칙
- SPEC/UI_SPEC/TASKS를 **절대 준수**한다.
- **디자인 임의 변경 금지** — UI_SPEC에 명시된 토큰·레이아웃·컴포넌트 규격만 사용.
- 모호/충돌 발견 시 **임의 결정 금지**, 반드시 이의 제기:
  1. 문제 보고 (어디가 왜)
  2. 선택지 2개 + 각 영향 분석
  3. 필요시 "가정 A로 진행한 구현안" 제시 (가정임을 명시)
- 산출물은 반드시 **재현 가능한 형태**로 제출 (파일 경로, diff, 실행 명령어, 기대 결과).

## 2. 운영 사이클 (기능 1개 단위)

```
┌─────────────────────────────────────────────────────────┐
│ Step 1  Claude: 다음 작업 선택 → SPEC/UI_SPEC/TASKS 업데이트    │
│ Step 2  Claude: ChatGPT용 프롬프트 생성 (PROMPTS.md 템플릿)     │
│ Step 3  사용자: 프롬프트를 ChatGPT에 전달                        │
│ Step 4  ChatGPT: 구현 산출물 제출 (재현 가능 형태)               │
│ Step 5  사용자: 결과 요약하여 Claude에 전달                      │
│         · 변경 파일 목록                                        │
│         · 핵심 변경 사항                                        │
│         · 실행 가능 여부                                        │
│         · ChatGPT 질문/이의 사항                                │
│ Step 6  Claude: 냉정 검수 → Must/Should/Nice 수정 지시          │
│ Step 7  사용자: 수정 지시를 ChatGPT에 전달                       │
│ Step 8  ChatGPT: diff 패치 제출 + 회귀 확인                     │
│ Step 9  Claude: Done 판단 → 다음 작업으로                       │
└─────────────────────────────────────────────────────────┘
```

### 2.1 사용자 전달 포맷

#### ChatGPT → Claude 결과 요약 템플릿
```markdown
## 작업 결과 보고: [작업명]

### 변경 파일
- `src/파일1.tsx` — 신규 생성
- `src/파일2.ts` — 수정 (함수 A 추가)

### 핵심 변경
- [변경 내용 1줄 요약]

### 실행 여부
- [ ] 로컬 빌드 성공
- [ ] 로컬 실행 확인
- [ ] 테스트 통과

### ChatGPT 질문/이의
- (있으면 기재)
```

## 3. 문서 세트 구성

| 파일 | 용도 | 변경 권한 |
|------|------|-----------|
| `PROJECT_PLAN.md` | 목표, MVP 범위, 스택, 마일스톤, 리스크 | Claude |
| `SPEC.md` | 데이터 모델, API, 인증, 에러정책, 폴더구조, 배포 | Claude |
| `UI_SPEC.md` | 디자인 토큰, 레이아웃, 컴포넌트, 반응형 | Claude |
| `TASKS.md` | 작업 목록, DoD, 검증 체크리스트 | Claude |
| `PROMPTS.md` | ChatGPT 고정 규칙 + 명령 템플릿 | Claude |

> ⛔ **ChatGPT는 위 문서들을 직접 수정하지 않는다.** 수정이 필요하면 이의 제기 후 Claude가 판단한다.

---

## 4. 검수 등급

| 등급 | 의미 | 대응 |
|------|------|------|
| **Must** | 반드시 수정. 미수정 시 작업 미완료 | ChatGPT 즉시 패치 |
| **Should** | 수정 권장. 다음 작업 전 처리 | ChatGPT 패치 후 확인 |
| **Nice** | 개선 제안. 현재 작업 범위 밖 가능 | 백로그 등록 |

---

## 5. 컨텍스트 동기화 규칙

1. **SSOT(Single Source of Truth)**: 모든 진실의 원천은 이 `_ai-collab/` 디렉토리 내 문서다.
2. Claude가 문서를 업데이트하면, 사용자가 최신 버전을 ChatGPT에 전달해야 한다.
3. ChatGPT가 구현 중 스펙 의문이 있으면, 해당 문서 섹션을 인용하여 질문한다.
4. **구두 합의 금지**: 모든 결정은 문서에 반영되어야 효력이 있다.
